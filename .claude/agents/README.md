# Subagents — 役割分担エージェント

このディレクトリには、Claude Code の Subagents 定義を置いています。
1つのエージェントに全部やらせるのではなく、**調査・レビュー・テスト設計** を専任エージェントに分業することで、指示の精度と出力品質を高めます。

---

## エージェント一覧

| エージェント | 役割 | 使う場面 |
|---|---|---|
| `repo-explorer` | 事前調査専任 | 実装・修正の前に関連箇所を把握したいとき |
| `reviewer` | 差分レビュー専任 | git diff をレビューしてほしいとき |
| `test-designer` | テスト設計専任 | テスト観点を整理してほしいとき（実装前） |

---

## 使い方

Claude Code のチャットで以下のように指示します。

```
# 事前調査
repo-explorer として、viewpoint_generation_service.py の責務と関連ファイルを調査してください。

# 差分レビュー
reviewer として、現在の git diff をレビューしてください。

# テスト設計
test-designer として、app/services/markdown_formatter.py のテスト観点を設計してください。
```

または `Agent` ツールを使って明示的に呼び出すこともできます。

---

## Skills との使い分け

このプロジェクトには Subagents のほかに Skills も導入済みです。

### Skills（`.claude/skills/`）

| Skill | 用途 |
|---|---|
| `/pytest-impl` | テストコードの実装（観点が固まってから使う） |

Skills は **定型タスクの入口** です。
「テスト観点は決まっているので、あとはコードを書くだけ」という状況で使います。

### Subagents（`.claude/agents/`）

Subagents は **役割分担された思考担当** です。
「まず何をすべきかを整理する」「差分を多角的にレビューする」という、判断・設計フェーズで使います。

### 典型的なワークフロー

```
1. repo-explorer → 関連箇所を把握
2. test-designer → テスト観点を設計
3. /pytest-impl  → テストコードを実装（Skill）
4. reviewer      → 実装差分をレビュー
```

---

## 設計の意図

- **1エージェント 1責務**: 調査が得意なエージェント・レビューが得意なエージェントを分ける
- **main context を汚さない**: 各エージェントは必要な情報だけを返す
- **Skills と相補的**: Skills が「実装」、Subagents が「設計・調査・レビュー」を担う
- **段階的な導入**: まず3役割から始め、必要に応じて追加できる土台にする
